# 아파치 카산드라
Apache Cassandra는 기존 관계형 데이터베이스(RDBMS)의 확장성 한계를 극복하기 위해 설계된 분산형 NoSQL 데이터베이스이다.
대규모 데이터를 여러 노드에 분산 저장하여 고가용성과 수평 확장성을 제공하는 것이 핵심이다.

## 데이터 모델
카산드라는 컬럼 패밀리 기반(column-family) 데이터베이스이다.
컬럼형 데이터베이스로 row를 기준으로 데이터가 추가되고 처리되는 것이 아닌, 컬럼을 기준으로 데이터를 저장한다.

## 노드와 클러스터
카산드라의 하나의 인스턴스를 노드(Node)라고 한다.
여러 노드가 모여 클러스터(Cluster)를 구성한다.
데이터는 여러 노드에 분산 저장 + 복제 된다.

## 확장성
카산드라의 가장 큰 장점은 수평 확장이다.
노드를 추가하면 자동으로 데이터가 분산되어 다운타임 없이 확장 가능하다.
대규모 트래픽에서도 안정적으로 처리가 가능하다.

## 쓰기 성능
카산드라는 쓰기 성능이 매우 빠르다.
일반 RDB 쓰기의 경우 랜덤 I/O라서 느리지만
카산드라는 무조건 뒤에 붙이는 append 방식이기 때문에 순차 I/O가 가능하다. (디스크에서 가장 빠른 방식) 

## 파티션 키
카산드라에서 가장 중요한 개념이다.
파티션 키는 데이터가 저장될 노드를 결정한다. (해시 기반으로 분산)
같은 파티션 키는 같은 노드에 저장되며 쿼리 시 반드시 파티션 키를 포함해야 효율적이다.
```sql
SELECT * FROM table WHERE partition_key = ?
```

## 데이터 설계
카산드라는 JOIN이 없기 때문에 데이터를 정규화된 테이블로 모델링 하는 것이 아니라
쿼리 구조에 맞춰 비정규화하며 대규모 환경에서 읽기 속도를 크게 향상시킨다.

## Cassandra vs MongoDB 
* Cassandra
  * 컬럼 기반 + 테이블 구조
  * 스키마 있음 (고정된 구조)
  * 쿼리에 맞춰 테이블을 설계 (비정규화 필수)
* MongoDB
  * 문서 기반
  * 스키마가 없음 (유연함)
  * 하나의 document에 관련 데이터 포함

카산드라는 어떤 조회를 할지 먼저 정의 후 조회 패턴별에 맞게 테이블을 따로 만든다.
MongoDB는 유연한 쿼리가 가능하며 복잡한 조회에 적합하다.
=> 카산드라는 쿼리 중심으로 설계된 수평 확장 가능한 분산 DB이며 정규화 대신 비정규화 + 파티션 키 설계가 성능을 좌우한다.
