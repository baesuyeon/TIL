Spring은 IoC를 DI 방식으로 구현한 프레임워크이다.

## IoC(제어의 역전)
IoC란 객체의 생성, 의존성 연결, 생명주기 관리에 대한 제어권을 
애플리케이션 코드가 아니라 프레임워크가 가져가는 것을 의미한다.

IoC가 없다면 필요한 객체를 주입해야하는데 `싱글톤`으로 관리하기도 어렵고 의존성에 대한 책임을 나누기가 어려워진다.
Spring에서는 IoC 컨테이너가 객체를 생성하고 의존성을 연결하고 싱글톤을 포함한 생명주기를 관리한다.

### IoC 컨테이너가 해주는 일
* 객체 생성
* 의존성 주입
* 싱글톤 관리
* 생명주기 콜백 (@PostConstruct, @PreDestroy)
* 프록시 생성 (AOP, 트랜잭션 등)

## DI(의존성 주입)
DI란 실행 시점(런타임)에 외부(Spring 컨테이너)에서 객체를 생성하여 
클라이언트가 필요로 하는 의존 객체를 주입해주는 방식이다.

클라이언트는
* 구현 클래스에 의존하지 않고
* 인터페이스에만 의존하며
* 어떤 구현체가 사용되는지 알지 못한다.

이로인해 클라이언트 코드를 변경하지 않고 설정만 바꿔서 객체간 의존관계를 유연하게 변경할 수 있다.

### 정리
* IoC는 개념
  * 제어권이 프레임워크로 넘어갔다는 설계 원칙
* DI는 구현 방법
  * IoC를 실현하는 대표적인 방법 중 하나
